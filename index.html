Problem Statement

The Clinical Documentation Gap

Clinicians spend significant time on manual documentation and textbook lookup, increasing cognitive load and reducing patient throughput.

No unified system integrates structured documentation, ML-based differential diagnosis, and validated medical textbook referencing in a single clinical workflow.

High Documentation Time

Manual case sheets average 12+ minutes per patient, reducing daily throughput.

Cognitive Overload

Physicians must recall differential diagnoses from memory without decision support.

Slow Textbook Lookup

Physical or digital textbook searches disrupt the patient consultation flow.

No Integrated System

Existing tools handle documentation OR diagnosis â€” never both in one workflow.

Model Selection

Why Decision Trees & Random Forest?

Chosen for interpretability, compatibility with sparse TF-IDF vectors, and strong multi-class performance.

Human-Readable Rules

Decision paths can be explained to clinicians â€” critical for medical AI trust.

Sparse TF-IDF Friendly

Handles high-dimensional sparse vectors (max_features=5000, ngram_range=(1,2)) natively.

No Feature Scaling Needed

Tree splits are threshold-based; no need for normalization or standardization.

Ensemble Reduces Variance

Bootstrap sampling + random feature subsets across 400 trees reduces overfitting dramatically.

Hyperparameters
n_estimators = 400 max_depth = None max_features = "sqrt" criterion = "gini" TF-IDF max_features = 5000 ngram_range = (1,2)
System Design

How CURA Works

Three integrated components handling input, prediction, and retrieval.

1

Symptom Input

Doctor enters symptoms as free text. TF-IDF vectorizer converts text into a 5,000-dimensional feature vector using 1â€“2 gram tokens.

2

RF Prediction

400 decision trees each predict a disease class. Majority vote determines the final differential diagnosis prediction.

3

RAG Retrieval

Based on the prediction, the RAG engine retrieves exact textbook page references and generates a structured case sheet.

How Random Forest Works in CURA

400 Diverse Trees

Each tree trains on a bootstrapped sample with random feature subsets (âˆš5000 â‰ˆ 71 features per split).

Majority Voting

Each tree votes for a disease class. The class with the most votes becomes the final prediction.

Gini Criterion

Each split minimises the weighted Gini impurity of child nodes, creating purer partitions.
